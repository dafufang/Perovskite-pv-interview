# Perovskite PV Interview Skill 使用说明

## 1. 这个 Skill 是做什么的？

`perovskite-pv-interview` 是一个面向钙钛矿光伏方向的双模式面试 Skill，既可以服务于候选人面试准备，也可以服务于公司招聘时评估候选人。

它覆盖的方向包括：

- 钙钛矿单结电池研发
- 钙钛矿/晶硅叠层电池
- 全钙钛矿叠层电池
- 器件物理与界面工程
- 薄膜沉积、涂布、蒸镀、溅射、ALD 等工艺
- 稳定性、可靠性和失效分析
- 封装、组件、互联和放大制造
- 设备、材料、工艺工程和技术负责人岗位

这个 Skill 的核心目标是把 AI 从“临时回答问题的通用助手”，转变成“按钙钛矿光伏面试 SOP 工作的专业助手”。

## 2. 进入 Skill 后的第一步

每次启动新的面试相关任务时，Skill 会先确认用户角色：

> 你是以候选人身份准备面试，还是以面试官/招聘方身份评估候选人？  
> Are you preparing as a candidate, or interviewing/evaluating a candidate?

确认角色后，Skill 会进入对应流程。

## 3. 两种使用模式

### 3.1 候选人模式

适用于以下场景：

- 我要准备钙钛矿研发岗位面试
- 这是我的简历，帮我预测面试官会问什么
- 帮我做一次英文 mock interview
- 帮我梳理我的项目经历怎么讲
- 我做过 p-i-n 钙钛矿电池，帮我准备技术追问

候选人模式下，Skill 会重点输出：

- 岗位能力画像
- 高频面试问题
- 简历项目追问预测
- 技术回答框架
- 中英文术语准备
- mock interview 问题
- 薄弱点复习清单
- 如何说明个人贡献、实验数据和技术机制

### 3.2 面试官 / 招聘方模式

适用于以下场景：

- 我要面试一个钙钛矿/晶硅叠层研发工程师
- 这是候选人简历，帮我设计面试问题
- 这是面试记录，帮我判断是否推荐
- 帮我设计稳定性工程师的评分表
- 帮我识别候选人简历中可能夸大的地方

面试官模式下，Skill 会重点输出：

- 面试结构和时间分配
- 岗位相关技术问题
- 针对简历经历的追问
- 项目真实性验证问题
- 强回答和弱回答信号
- 红旗信号
- 评分 rubric
- 推荐 / 谨慎推荐 / 不推荐等录用建议

## 4. 上传附件简历分析功能

这个 Skill 支持用户上传简历、CV、项目经历、论文列表或类似附件作为输入。

当用户上传简历后，Skill 会先读取附件内容，再生成结构化分析，而不是直接泛泛评价。

简历分析通常包括：

1. **候选人画像**  
   教育背景、经验年限、技术方向、岗位匹配度和可能的职级。

2. **项目经历梳理**  
   按每段经历拆解项目目标、技术路线、候选人声称的贡献、结果和证据。

3. **技术关键词提取**  
   包括器件结构、传输层、添加剂、钝化、制备工艺、表征方法、稳定性测试、组件放大、设备和论文/专利信息。

4. **缺失信息识别**  
   例如器件面积、效率是否认证、是否有稳态输出、是否有统计数据、稳定性条件、个人贡献边界、良率和放大规模等。

5. **风险与验证点**  
   识别表述过于笼统、成果归属不清、缺少数据支撑、只讲 champion device、缺少稳定性或放大经验等问题。

然后根据用户角色分流：

- **候选人模式**：根据简历预测面试问题、梳理回答框架、找薄弱点、生成 mock interview。
- **面试官模式**：根据简历生成追问清单、真实性验证问题、评分重点、红旗信号和录用评估模板。

Skill 会明确区分：

- 简历中明确写到的信息
- AI 基于简历做出的合理推断
- 面试中需要进一步确认的问题

## 5. 使用 Skill 和不使用 Skill 的区别

### 5.1 不使用这个 Skill 时

AI 更像是一个通用助手。你问：

> 帮我分析这个钙钛矿方向候选人的简历。

它通常会直接回答，但输出质量会更依赖你当时的问题写法。可能会出现这些情况：

- 会分析简历，但不一定先问你是候选人还是面试官
- 可能泛泛总结候选人经历，而不是按钙钛矿岗位能力维度拆解
- 可能给出一些面试问题，但追问逻辑不一定系统
- 可能忽略钙钛矿行业里很关键的验证点，例如：
  - champion efficiency 和统计效率是否区分
  - 小面积器件和大面积组件是否区分
  - J-V 扫描效率和稳态输出是否区分
  - EQE 积分电流是否匹配
  - 稳定性测试条件是否充分
  - 候选人个人贡献和团队贡献是否分清
- 输出格式每次可能不同
- 对候选人准备和公司招聘评估的角度可能混在一起

也就是说，不用 Skill 时，AI 可以回答，但更依赖用户每次把需求描述得足够完整。

### 5.2 使用这个 Skill 时

AI 会被带入一个固定的专业流程。

这个 Skill 会让 AI 在开始时先做一件关键事：

> 先判断或询问用户角色：你是候选人，还是面试官/招聘方？

然后再进入对应流程。

#### 如果用户是候选人

AI 会更像一个钙钛矿光伏面试教练，重点帮助用户：

- 根据简历预测面试官会问什么
- 梳理项目经历的表达逻辑
- 找出简历里的技术薄弱点
- 准备中英文面试回答
- 模拟技术面试
- 把经历讲成“问题—方法—数据—结果—个人贡献”
- 提醒哪些地方容易被追问

例如用户写：

> 我做过 p-i-n 钙钛矿电池效率优化。

Skill 会进一步拆解：

- 优化的是哪个层：吸收层、界面层、传输层还是电极？
- 效率提升主要来自 Voc、Jsc 还是 FF？
- 有没有对照实验？
- 数据是否有统计重复性？
- 是否有稳态输出或 MPPT？
- 个人负责的是实验设计、制备、表征、数据分析，还是论文写作？

#### 如果用户是面试官 / 招聘方

AI 会更像一个专业技术面试官助手，重点帮助用户：

- 根据岗位生成面试提纲
- 根据简历设计追问
- 判断候选人经历真实性
- 区分候选人个人贡献和团队成果
- 设计评分表
- 识别红旗信号
- 给出是否推荐、适合什么岗位、后续还要验证什么

例如候选人写：

> 负责钙钛矿/晶硅叠层电池效率提升。

Skill 会引导 AI 追问：

- 叠层结构是两端还是四端？
- 顶电池带隙是多少？
- 电流匹配如何判断？
- 中间连接层 / 复合层如何设计？
- EQE 积分电流和 J-V 电流是否一致？
- 效率是否第三方认证？
- 面积是多少？
- 稳态效率和扫描效率是否一致？
- 候选人个人负责的是哪一部分？

### 5.3 对比表

| 对比项 | 不使用 Skill | 使用 Skill |
|---|---|---|
| 角色识别 | 可能不问 | 先问候选人/面试官 |
| 输出流程 | 依赖用户临时描述 | 固定进入对应面试流程 |
| 专业维度 | 可能泛化 | 聚焦钙钛矿光伏岗位能力 |
| 简历分析 | 总结为主 | 总结 + 追问 + 风险点 + 准备建议 |
| 面试问题 | 可能零散 | 按岗位、难度、技术方向组织 |
| 追问能力 | 不一定深入 | 强制生成技术追问逻辑 |
| 评分标准 | 可能临时生成 | 使用固定 rubric |
| 红旗识别 | 可能遗漏 | 明确检查效率、稳定性、放大、贡献归属等问题 |
| 中英支持 | 需要用户提醒 | 默认支持中英双语 |
| 一致性 | 每次可能不同 | 输出更稳定、可复用 |

一句话总结：

> 不用 Skill，AI 是“临时帮你分析”。  
> 使用 Skill，AI 是“按照一套钙钛矿光伏面试 SOP 来工作”。

## 6. 推荐使用方式

### 候选人准备示例

```text
我是候选人，目标岗位是钙钛矿/晶硅叠层研发工程师。我上传了简历，请帮我分析面试官可能会问的问题，并给出回答框架。
```

```text
我是候选人，请用英文模拟一个 30 分钟钙钛矿器件研发岗位技术面试。先问一个问题，等我回答后再反馈。
```

### 面试官评估示例

```text
我是面试官，要面试一个 3-5 年经验的钙钛矿稳定性工程师。我上传了候选人简历，请帮我设计 45 分钟技术面试提纲和追问问题。
```

```text
我是招聘方，这是候选人的面试记录。请根据钙钛矿/晶硅叠层高级研发工程师岗位给出评分、红旗信号和录用建议。
```

### 双语输出示例

```text
我是候选人，准备英文面试。请根据我的中文简历生成英文 mock interview 问题，并给出中英文术语对照。
```

## 7. 输出原则

Skill 在输出时应遵循以下原则：

- 先明确用户角色，再进入流程
- 尽量基于简历、JD、面试记录或用户提供的信息分析
- 不编造简历中没有的信息
- 区分事实、推断和待验证问题
- 对技术问题给出追问逻辑，而不仅仅是题目
- 对候选人模式强调准备和表达
- 对面试官模式强调验证、评分和风险判断
- 对最新效率纪录、公司动态、当前产业信息等时效性内容，应检索最新来源后再回答

## 8. PDF 报告输出功能

这个 Skill 可以把最终结果整理成 PDF 报告，适合用于候选人面试准备留档、招聘评估汇报、面试后复盘或内部讨论。

当用户提出以下需求时，应生成报告型输出，并在支持文件生成的环境中导出为 PDF：

- “请输出一个 PDF 报告”
- “帮我生成候选人评估报告”
- “把这份简历分析整理成报告”
- “面试后给我一个 hiring memo”
- “生成 mock interview feedback report”

PDF 报告最终参考三个固定框架，根据用户不同需求自动选择。

### 8.1 候选人面试准备报告

适用于用户是候选人，例如：

> 我准备面试钙钛矿/晶硅叠层研发岗，这是我的简历，帮我生成准备报告。

推荐文件名：

```text
perovskite_interview_prep_report.pdf
```

报告结构：

1. Executive Summary / 摘要
   - 目标岗位
   - 候选人背景概览
   - 面试准备重点
   - 主要优势
   - 主要风险或短板

2. Candidate Profile / 候选人画像
   - 教育背景
   - 研究方向
   - 项目经历
   - 论文 / 专利 / 奖项
   - 设备、工艺、表征技能
   - 与目标岗位的匹配度

3. Target Role Analysis / 目标岗位分析
   - 岗位核心能力要求
   - 必备技术能力
   - 加分能力
   - 可能被重点考察的方向

4. Resume-Based Interview Question Forecast / 基于简历的面试问题预测
   - 针对每段项目经历的问题
   - 针对论文 / 专利的问题
   - 针对实验细节的问题
   - 针对个人贡献的问题
   - 针对稳定性、放大、封装、叠层等方向的问题

5. Technical Deep-Dive Topics / 技术深挖主题
   - 钙钛矿器件结构
   - p-i-n / n-i-p
   - 界面工程
   - 缺陷钝化
   - 薄膜结晶
   - 稳定性
   - 封装
   - 钙钛矿/晶硅叠层
   - 量产与放大

6. Suggested Answer Frameworks / 答题框架
   - 项目经历回答框架
   - 技术问题回答框架
   - 失败案例回答框架
   - 个人贡献说明框架
   - 英文回答建议

7. Weakness and Risk Areas / 薄弱点与风险点
   - 简历中未充分体现的能力
   - 可能被追问但准备不足的内容
   - 容易被质疑的表述
   - 建议补充准备的知识点

8. Practice Questions / 练习问题
   - 基础问题
   - 中级问题
   - 高级问题
   - 情景题 / 实验设计题
   - 英文 mock interview 问题

9. Preparation Checklist / 面试前准备清单
   - 必须复习的技术点
   - 必须准备的数据
   - 必须讲清楚的个人贡献
   - 建议准备的图表或案例

10. Appendix / 附录
   - 中英术语表
   - 技术关键词表
   - 推荐追问清单

### 8.2 面试官候选人评估报告

适用于用户是面试官或招聘方，例如：

> 这是候选人简历，帮我判断是否适合高级钙钛矿叠层研发工程师岗位，并输出 PDF 报告。

推荐文件名：

```text
perovskite_candidate_evaluation_report.pdf
```

报告结构：

1. Executive Summary / 摘要
   - 候选人整体判断
   - 目标岗位
   - 推荐结论
   - 主要优势
   - 主要风险
   - 是否建议进入下一轮

2. Candidate Background Summary / 候选人背景摘要
   - 教育与工作经历
   - 核心项目经历
   - 技术方向
   - 论文 / 专利 / 成果
   - 关键技能标签

3. Role Fit Assessment / 岗位匹配度评估
   - 与目标岗位的匹配程度
   - 已体现能力
   - 未充分体现能力
   - 需要面试验证的能力

4. Technical Competency Evaluation / 技术能力评估
   - 器件物理理解
   - 薄膜制备与工艺能力
   - 界面工程与缺陷控制
   - 表征与数据分析能力
   - 稳定性与可靠性意识
   - 叠层电池理解
   - 放大与量产意识
   - 封装与组件经验

5. Resume-Based Probing Questions / 基于简历的追问问题
   - 项目真实性验证问题
   - 个人贡献验证问题
   - 数据可靠性验证问题
   - 技术路线选择问题
   - 失败案例与问题诊断问题

6. Suggested Interview Plan / 建议面试流程
   - 30 分钟版本
   - 45 分钟版本
   - 60 分钟版本
   - 每一阶段建议提问方向

7. Scoring Rubric / 评分表
   - 基础理论
   - 实验能力
   - 问题诊断
   - 工艺放大
   - 稳定性意识
   - 产业化判断
   - 沟通与归因
   - 综合评分

8. Red Flags / 红旗信号
   - 只讲 champion efficiency，不讲统计数据
   - 不区分小面积器件与大面积组件
   - 不区分 J-V 扫描效率与稳态输出
   - 稳定性测试条件含糊
   - 个人贡献与团队贡献不清
   - 叠层经验表述不具体
   - 缺少对失败实验的复盘能力

9. Hiring Recommendation / 录用建议
   - 强烈推荐 / 推荐 / 有保留 / 不推荐
   - 推荐岗位方向
   - 建议职级
   - 入职后可能需要补强的能力
   - 下一轮面试建议

10. Appendix / 附录
   - 面试问题清单
   - 评分细则
   - 技术术语表
   - 简历信息提取表

### 8.3 Mock Interview Feedback 报告

适用于用户做模拟面试后，例如：

> 你刚才模拟面试了我，请根据我的回答生成一份反馈 PDF。

推荐文件名：

```text
perovskite_mock_interview_feedback.pdf
```

报告结构：

1. Executive Summary / 摘要
   - 模拟面试岗位
   - 总体表现
   - 推荐提升方向
   - 主要优势
   - 主要短板

2. Interview Context / 面试背景
   - 目标岗位
   - 面试语言
   - 面试难度
   - 重点技术方向
   - 本次模拟覆盖的问题类型

3. Question-by-Question Feedback / 逐题反馈
   - 原问题
   - 候选人回答摘要
   - 回答优点
   - 回答不足
   - 建议优化回答
   - 可被继续追问的点

4. Technical Accuracy Review / 技术准确性检查
   - 概念是否准确
   - 逻辑是否完整
   - 数据表达是否清楚
   - 是否体现个人贡献
   - 是否能解释机制与证据

5. Communication Assessment / 表达能力评估
   - 结构清晰度
   - 英文表达能力，如适用
   - 技术叙述深度
   - 面试官友好度
   - 回答是否过于泛泛

6. Competency Scorecard / 能力评分卡
   - 专业基础
   - 项目表达
   - 技术深度
   - 问题诊断
   - 产业化理解
   - 面试沟通
   - 综合准备度

7. Improved Answer Examples / 优化回答示例
   - 中文优化版
   - 英文优化版，如需要
   - STAR / Problem-Method-Result-Contribution 框架

8. Follow-Up Practice Plan / 后续练习计划
   - 接下来 3 天复习重点
   - 接下来 1 周练习重点
   - 需要补充准备的数据、图表或案例
   - 推荐继续 mock 的问题

9. Appendix / 附录
   - 本次模拟问题列表
   - 术语表
   - 高频追问问题

### 8.4 报告输出原则

PDF 报告应明确区分：

- 用户提供的事实，例如简历内容、JD、面试记录
- AI 的分析和判断
- 仍需面试确认的问题

报告应避免把推断写成确定事实。对于钙钛矿光伏候选人，报告尤其要检查效率数据、器件面积、稳态输出、稳定性条件、统计重复性、放大经验和个人贡献边界。

## 9. 增强版优化内容

本版本新增了一组面向真实招聘与候选人准备的增强模块，使 Skill 更像标准化技术面试系统，而不只是题库。

### 9.1 岗位等级分层

Skill 会根据目标岗位和候选人背景区分不同面试深度：

- 实习 / 应届
- 硕士 / 博士候选人
- 1-3 年经验工程师
- 3-7 年经验工程师
- 高级 / Staff / Principal 工程师
- Manager / Director / 技术负责人

不同职级对应不同考察重点。例如，初级候选人重点看基础和诚实表达；中级候选人重点看独立实验、数据分析和问题诊断；高级候选人重点看路线判断、工艺放大、稳定性和产业化权衡；负责人重点看技术路线、资源配置、团队建设和风险管理。

### 9.2 岗位画像模板

Skill 新增了多类钙钛矿光伏岗位画像，用于把简历、JD 或面试目标映射到具体能力要求：

- 钙钛矿单结研发工程师
- 钙钛矿/晶硅叠层工程师
- 全钙钛矿叠层工程师
- 稳定性与可靠性工程师
- 封装与组件工程师
- 薄膜 / 工艺工程师
- 设备工程师
- 材料研发工程师
- 技术负责人 / 项目负责人

每个岗位画像包含核心职责、必备技能、加分项、关键追问方向和不匹配信号。

### 9.3 简历真实性验证模块

当用户上传简历或提供项目经历时，Skill 会把候选人主张拆成可验证问题，并区分：

- 明确事实：简历或用户明确提供的信息
- 合理推断：基于上下文可以推测但未直接写明的信息
- 待验证事项：需要面试追问确认的信息
- 证据不足：不能作为判断依据的信息

重点验证内容包括：

- 效率是否说明面积、统计、稳态输出、EQE 匹配和认证情况
- 叠层经验是否说明 2T/4T、电流匹配、TCO、复合层和限流子电池
- 稳定性是否说明测试条件、封装状态、MPPT/SPO、样本数量和退化机制
- 放大经验是否说明面积、均匀性、良率、节拍、缺陷和工艺窗口
- 个人贡献是否与团队成果区分清楚

### 9.4 面试追问树

Skill 不只生成单个问题，还会针对关键技术点生成分层追问树。例如：

- 界面钝化：钝化哪个界面、什么缺陷、什么证据、是否有副作用、是否可放大
- 薄膜结晶：沉积方法、成核/生长控制、形貌证据、工艺窗口和失效模式
- 叠层电池：2T/4T、带隙、电流匹配、EQE 积分电流、透明电极和复合层
- 稳定性：测试条件、封装状态、MPPT/SPO、T80/T95、失效机理
- 放大制造：面积、均匀性、良率、缺陷、节拍、在线检测和成本权衡

### 9.5 评分校准标准

Skill 新增 1-5 分评分锚点，帮助不同面试官获得更一致的评分结果。

示例：稳定性意识评分

- 1 分：只知道稳定性重要，但说不清测试条件
- 3 分：能说明光照、热、湿热、偏压、封装状态、MPPT/SPO 和样本数量等基本要素
- 5 分：能系统区分失效机制，并把界面、封装、离子迁移、可靠性测试和产品要求联系起来

评分报告会尽量说明证据和置信度，避免把未验证信息当作确定结论。

### 9.6 中英术语与回答模板

Skill 新增中英文术语和回答模板，适合英文面试、海外候选人、国际合作团队或候选人准备英文 mock interview。

覆盖内容包括：

- 钙钛矿光伏核心术语中英对照
- 项目经历回答模板
- 失败案例回答模板
- 个人贡献说明模板
- 面试官英文追问句式

### 9.7 PDF 报告质量检查

在生成最终 PDF 报告前，Skill 会参考质量检查清单，确认：

- 是否选择了正确报告模板
- 是否明确用户角色、目标岗位和职级
- 是否区分事实、推断和待验证问题
- 是否包含钙钛矿专业检查项
- 是否包含简历真实性验证
- 是否包含校准评分和置信度
- 是否包含下一步行动建议
- 中文、英文、表格和分页是否适合 PDF 输出
