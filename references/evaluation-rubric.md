# Evaluation rubric and hiring memo

Use a 1-5 score. Anchor every score in observed evidence.

| Dimension | 1-2 weak | 3 acceptable | 4-5 strong |
|---|---|---|---|
| Fundamentals | Confuses stack, metrics, or mechanisms | Understands standard concepts | Explains mechanisms quantitatively and links them to data |
| Hands-on ownership | Vague about personal contribution | Can describe tasks performed | Owns design, controls, execution, and interpretation |
| Data rigor | Relies on champion results | Provides some repeatability | Gives statistics, controls, area, scan protocol, MPPT/EQE consistency |
| Problem diagnosis | Generic explanations | Can list plausible causes | Builds prioritized hypotheses and verification experiments |
| Stability mindset | Treats stability as afterthought | Knows common tests | Designs test matrix and separates degradation mechanisms |
| Scale-up/manufacturing | Lab-only view | Understands some scale issues | Discusses yield, uniformity, takt, process windows, QC, cost |
| Tandem expertise if relevant | Uses buzzwords | Understands current matching and stack basics | Can analyze subcell limitation, optics, TCO/contact damage, certification evidence |
| Communication | Unclear or overclaims | Mostly clear | Precise, honest, evidence-backed, distinguishes personal vs team work |

## Hiring recommendation labels

- **Recommend / 推荐**: clear evidence of role-relevant depth and ownership; risks are manageable.
- **Lean recommend / 倾向推荐**: strong enough for role, with one or two gaps that can be managed.
- **Hold / 待定**: mixed evidence; needs another interview, technical exercise, or reference check.
- **Lean no / 倾向不推荐**: important gaps for the target role.
- **No / 不推荐**: lacks required fundamentals, ownership, rigor, or integrity.
